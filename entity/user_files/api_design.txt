To improve your API design for managing deployment and environment configuration, we'll adjust the endpoints and requests to adhere to RESTful principles. This means using standard HTTP methods and making sure that the endpoints clearly represent the resources being manipulated.

Here's a revised outline of your APIs:

### API Overview

1. **Authentication**
   - `Authorization: Bearer <token>` for all requests

2. **Environment Management APIs**
   
   - **POST /deploy/cyoda-env**
     - **Description:** Initiate a build for Cyoda environment configuration.
     - **Request Body:**
       ```json
       {
         "user_name": "test"
       }
       ```
     - **Action:** 
       ```http
       POST https://teamcity.cyoda.org/app/rest/buildQueue
       {
         "buildType": {
           "id": "KubernetesPipeline_CyodaSaas"
         },
         "properties": {
           "property": [
             {
               "name": "user_defined_keyspace",
               "value": "test"
             },
             {
               "name": "user_defined_namespace",
               "value": "test"
             }
           ]
         }
       }
       ```
     - **Response:** Returns build ID

   - **GET /deploy/cyoda-env/status/{build_id}**
     - **Description:** Retrieve the status of a specific Cyoda environment deployment.
     - **Action:** 
       ```http
       GET https://teamcity.cyoda.org/app/rest/buildQueue/id:{build_id}
       ```
     - **Response:** Status information about the build

   - **GET /deploy/cyoda-env/statistics/{build_id}**
     - **Description:** Retrieve statistics for a specific Cyoda environment deployment.
     - **Action:** 
       ```http
       GET https://teamcity.cyoda.org/app/rest/builds/id:{build_id}/statistics/
       ```
     - **Response:** Statistics related to the build

3. **User Application APIs**
   
   - **POST /deploy/user_app**
     - **Description:** Initiate a build for deploying a user application.
     - **Request Body:**
       ```json
       {
         "repository_url": "http://....",
         "is_public": "true"
       }
       ```
     - **Action:** 
       ```http
       POST https://teamcity.cyoda.org/app/rest/buildQueue
       {
         "buildType": {
           "id": "KubernetesPipeline_CyodaSaasUserEnv"
         },
         "properties": {
           "property": [
             {
               "name": "user_defined_keyspace",
               "value": "user_name"
             },
             {
               "name": "user_defined_namespace",
               "value": "user_name"
             }
           ]
         }
       }
       ```
     - **Response:** Returns build ID
     
   - **GET /deploy/user_app/status/{build_id}**
     - **Description:** Retrieve the status of a specific user application deployment.
     - **Action:** 
       ```http
       GET https://teamcity.cyoda.org/app/rest/buildQueue/id:{build_id}
       ```
     - **Response:** Status information about the user app build

   - **GET /deploy/user_app/statistics/{build_id}**
     - **Description:** Retrieve statistics for a specific user application deployment.
     - **Action:** 
       ```http
       GET https://teamcity.cyoda.org/app/rest/builds/id:{build_id}/statistics/
       ```
     - **Response:** Statistics related to the user app build

   - **POST /deploy/cancel/user_app/{build_id}**
     - **Description:** Cancel a queued user application build.
     - **Action:** 
       ```http
       POST https://teamcity.cyoda.org/app/rest/builds/id:{build_id}
       {
         "comment": "Canceling a queued build",
         "readdIntoQueue": false
       }
       ```
     - **Response:** Confirmation of cancellation

### Key Improvements in RESTful Design:
- **Resource Identification:** The endpoint names are rewritten to clearly indicate what resource is being acted upon, providing better context.
- **Nouns vs. Verbs:** APIs should use nouns representing resources rather than verbs and actions within the path, which is better aligned with RESTful principles.
- **Consistent Endpoint Structure:** All endpoints follow a similar structure, making it easier for users to remember and use the APIs.
- **Use of Proper HTTP Methods:** Distinct HTTP methods (GET, POST) are used for retrieving vs. creating resources.

By implementing these changes, your API will follow RESTful conventions, making it more intuitive for developers to use and maintain.
