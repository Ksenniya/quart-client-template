Here is the outline of user APIs for each API_REQUEST type entity, specifying the entity name and related API endpoints along with descriptions and pseudocode.

### 1. User Environment Entity

**API Endpoint:** `POST /deploy/cyoda-env`

**Description:**
- This endpoint creates or updates the user environment configuration.
- **Pseudocode:**
  ```plaintext
  # Validate user input
  # Create or update the environment configuration
  save_environment(user_name, configuration)
  # Trigger workflow event for environment setup
  trigger_workflow(user_name, environment_id)
  return environment details to user
  ```

**API Endpoint:** `PUT /deploy/cyoda-env/{id}`

**Description:**
- This endpoint updates an existing user environment configuration.
- **Pseudocode:**
  ```plaintext
  # Validate user input
  # Update the environment configuration by ID
  update_environment(id, new_configuration)
  return updated environment details to user
  ```

**API Endpoint:** `GET /deploy/cyoda-env/{id}`

**Description:**
- This endpoint retrieves the details of a specific user environment configuration.
- **Pseudocode:**
  ```plaintext
  # Fetch environment by ID from Cyoda
  environment = fetch_environment_by_id(id)
  return environment details to user
  ```

### 2. Deployment Entity

**API Endpoint:** `POST /deploy/user_app`

**Description:**
- This endpoint initiates the deployment process for a user application using a repository URL.
- **Pseudocode:**
  ```plaintext
  # Validate user input
  # Save deployment request and trigger workflow
  save_deployment(repository_url, is_public)
  trigger_workflow(deployment_id)
  return deployment details to user
  ```

**API Endpoint:** `PUT /deploy/user_app/{id}`

**Description:**
- This endpoint updates the details of an existing application deployment.
- **Pseudocode:**
  ```plaintext
  # Validate user input
  # Update deployment details by ID
  update_deployment(id, updated_data)
  return updated deployment details to user
  ```

**API Endpoint:** `GET /deploy/user_app/{id}`

**Description:**
- This endpoint retrieves the status of a specific application deployment.
- **Pseudocode:**
  ```plaintext
  # Fetch deployment by ID from Cyoda
  deployment = fetch_deployment_by_id(id)
  return deployment details to user
  ```

### 3. Environment Status Entity

**API Endpoint:** `GET /deploy/cyoda-env/status/{id}`

**Description:**
- This endpoint fetches the status of a deployed environment.
- **Pseudocode:**
  ```plaintext
  # Fetch environment status by deployment ID from Cyoda
  status = fetch_environment_status_by_id(id)
  return environment status details to user
  ```

### 4. Cancellation Entity

**API Endpoint:** `POST /deploy/cancel/user_app/{id}`

**Description:**
- This endpoint requests the cancellation of a queued deployment.
- **Pseudocode:**
  ```plaintext
  # Validate cancellation request
  # Process cancellation of deployment by ID
  cancel_deployment(id, comment)
  return cancellation confirmation to user
  ```

**API Endpoint:** `GET /deploy/cancel/user_app/{id}`

**Description:**
- This endpoint retrieves the cancellation status of a deployment.
- **Pseudocode:**
  ```plaintext
  # Fetch cancellation status by deployment ID from Cyoda
  cancellation_status = fetch_cancellation_status_by_id(id)
  return cancellation status details to user
  ```

### Summary
Each endpoint is tailored to the relevant entity, providing a clear structure for how to manage and retrieve information related to user environments and deployments. The pseudocode gives a simplified representation of the logic behind the endpoints, ensuring clarity on what operations will be performed.
